# Workspace 回归清单（V1）

适用范围：`croupier-dashboard` 对象工作台（Workspaces / WorkspaceEditor / Console）。

## 1. 角色维度

- [ ] **管理员**：可查看列表、编辑、保存、发布、取消发布、查看版本、回滚。
- [ ] **编辑者（无发布权限）**：可编辑与保存；发布/取消发布/回滚按钮不可用或不可见。
- [ ] **只读用户**：可查看控制台已发布内容；编辑入口不可用。
- [ ] **无权限用户**：访问管理页或详情页时收到明确错误提示，不出现静默失败。

## 2. 页面维度

### 2.1 Workspaces 列表页

- [ ] 首次加载成功，展示标题、描述、状态、版本、更新时间。
- [ ] 搜索（标题/objectKey/描述）生效，空结果出现空态提示。
- [ ] 状态筛选（全部/草稿/已发布/已归档）生效。
- [ ] 排序（更新时间降序/标题升序）生效。
- [ ] 已归档配置不允许编辑、发布、取消发布。
- [ ] 加载失败时展示错误态，不出现白屏。

### 2.2 WorkspaceEditor 编辑页

- [ ] `objectKey` 非法时给出错误提示。
- [ ] 可新增/编辑/删除/排序 Tab，且默认 Tab 规则始终合法。
- [ ] 仅允许 `tabs` 顶层布局与 `form-detail/list/form/detail` 子布局。
- [ ] 保存前校验可拦截非法配置，并输出可读错误列表。
- [ ] 模板应用仅允许 V1 支持布局，非法模板被拦截。
- [ ] 版本面板可加载版本列表，回滚前显示差异摘要与风险提示。

### 2.3 Console 列表页

- [ ] 仅展示已发布配置。
- [ ] 搜索/排序生效。
- [ ] 无数据时展示空态，失败时展示错误态。
- [ ] 点击卡片可进入详情页。

### 2.4 Console 详情页

- [ ] `objectKey` 为空或非法时展示错误提示。
- [ ] 配置不存在时展示错误提示。
- [ ] 渲染器与编辑器预览一致（同配置下展示一致）。
- [ ] 非 V1 支持布局时展示明确错误，不出现“假可用”状态。

## 3. 发布链路维度

- [ ] 新建配置后可保存草稿并再次打开。
- [ ] 发布成功后，Workspaces 状态与 Console 可见性同步正确。
- [ ] 取消发布后，Console 侧不可见，Workspaces 状态回落正确。
- [ ] 发布/取消发布确认弹窗显示对象、标题、关键摘要。
- [ ] 回滚后可再次保存与发布，历史版本仍可查询。

## 4. 治理与可观测性

- [ ] 权限点生效：`read/edit/publish/rollback/delete` 与 UI 操作一致。
- [ ] 关键埋点可观测：页面打开、加载、保存、发布、取消发布、回滚、渲染失败。
- [ ] 渲染失败或加载失败时，用户可见错误提示，且不阻塞页面其他基础交互。

## 5. 执行说明

- [ ] 回归执行顺序：先角色权限，再页面功能，再发布链路，最后版本回滚。
- [ ] 每条失败项必须记录：角色、页面、对象 `objectKey`、操作步骤、错误截图、控制台日志。
- [ ] 回归通过标准：P0/P1 问题为 0，P2 可带风险评估进入下一轮。
